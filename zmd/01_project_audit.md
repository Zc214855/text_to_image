# 阶段 1：项目审计

## 已确认问题

1. `.env.example` 包含真实 API Key，且智谱变量不符合 dotenv 命名规范，火山配置不是合法键值对。
2. 智谱配置使用 Anthropic 协议地址，但项目只使用 OpenAI SDK，协议不兼容。
3. 火山方舟仅提供 API Key，缺少调用必需的模型 ID 或 Endpoint ID。
4. `get_model_config()` 读取启动时的 `IMAGE_MODEL`，界面切换模型后可能继续使用旧模型参数。
5. 分镜提示词重复角色描述和风格后缀，浪费上下文并降低模型对动作、构图的关注。
6. 只固定重复角色，妈妈、奶奶、猎人等一次性角色在不同画面中没有稳定设定。
7. 按约 40 字强制切分导致镜头过碎，部分场景缺少完整动作和视觉主体。
8. 没有系统化的镜头景别、构图、时间、色板约束，整套插图缺少连续性。
9. “no text” 仅作为普通正向提示，样例图片仍生成了路牌文字。
10. 图片下载未校验响应类型和文件内容，错误页面可能被保存为 `.png`。
11. `start.bat` 硬编码外部项目盘符和特定用户 Python 路径，无法跨机器运行。
12. 项目没有自动化测试。

## 样例输出观察

- 角色主体画风基本统一，但镜头内容与故事动作匹配度不足。
- 场景 1 出现带文字路牌，违反无文字要求。
- 场景 11 未表现“大灰狼伪装成奶奶躺在床上”，说明提示词丢失关键叙事状态。
- 多个镜头重复采用正面双人站立构图，缺乏远景、全景、中景和特写变化。
