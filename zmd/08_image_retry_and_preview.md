# 阶段 8：图片重试与 1K 预览

## 图片自动重试

1. 每张图片默认最多尝试 3 次，即首次请求加 2 次自动重试。
2. 仅对明确返回的 HTTP 408/409/425/429 和 5xx 服务错误自动重新提交。
3. POST 网络异常或读取超时不自动重新提交，避免服务端已受理时重复计费。
4. 图片下载失败只重试同一个结果 URL，不会重新提交生图请求。
5. 鉴权、参数、余额、审核等永久错误不自动循环请求。
6. 重试等待时间依次为 1 秒、2 秒。

## 失败任务持久化

每个故事目录新增 `generation.json`，记录：

- 图片模型和尺寸。
- 图片扩展名。
- 总场景数。
- 已成功场景。
- 失败场景及错误信息。
- 自动重试次数。
- 任务状态和更新时间。

状态在每张图片结束后立即写入。程序中断后，页面重新打开仍可发现缺失图片并继续。

## 仅重试图片

新增“仅重试失败图片（不调用 LLM）”按钮：

1. 读取原故事目录的 `prompts.json`。
2. 使用 `generation.json` 中保存的原图片模型和尺寸。
3. 只生成失败或文件缺失的场景。
4. 保留所有已成功图片。
5. 当前页面没有任务状态时，自动查找最近的未完成任务。

## Seedream 4.0

新增 `doubao-seedream-4-0-250828`：

- 支持 1024x1024、864x1152、1152x864 等 1K 尺寸。
- 定位为低分辨率构图预览。
- 明确请求 PNG 输出，与任务文件扩展名保持一致。

## 界面验证

1. Seedream 4.0 可在图像模型下拉框中正常选择。
2. 切换后默认尺寸自动更新为 `1024x1024`。
3. 模型 ID、用途和可选尺寸说明同步更新。
4. 静态链路说明明确标记为默认启动配置，避免与页面当前选择混淆。
